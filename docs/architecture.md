# Архитектура учебной конфигурации

## Основные блоки

### НСИ

Блок нормативно-справочной информации содержит справочники для хранения сотрудников, должностей, номенклатуры, единиц измерения, складов, организаций, контрагентов, договоров, валют и видов цен.

### Документооборот

Документы фиксируют хозяйственные операции:

- `ПриобретениеТоваров` — поступление товаров на склад;
- `РеализацияТоваров` — отгрузка товаров покупателю;
- `УстановкаЦенНоменклатуры` — регистрация цен номенклатуры.

### Регистры

Регистры используются для хранения итоговых и исторических данных:

- `ОстаткиТоваров` — количественный учет остатков;
- `СтоимостьТоваров` — суммовой учет стоимости остатков;
- `Продажи` — обороты продаж;
- `ЦеныНоменклатуры` — цены по видам цен и валютам;
- `КурсыВалют` — курсы валют по датам;
- `Управленческий` — управленческие бухгалтерские проводки.

### Отчетность

Отчеты строятся на запросах, виртуальных таблицах регистров и СКД. Основные отчеты: остатки товаров на складах и продажи.

### Администрирование

Реализованы роли пользователей, параметры сеанса, константы и использование журнала регистрации.

## Логика проведения

При проведении документов формируются движения по регистрам:

- поступление увеличивает остатки и стоимость товаров;
- реализация уменьшает остатки и стоимость, а также отражает продажи;
- установка цен записывает данные в регистр сведений цен.

## Особенности проекта

Проект учебный, поэтому часть решений сделана для демонстрации механизмов платформы, а не как полностью промышленная складская система.
