# Инструкция по загрузке конфигурации

## Вариант 1. Загрузка `.cf`

1. Открыть 1С:Предприятие.
2. Создать новую пустую информационную базу.
3. Запустить ее в режиме **Конфигуратор**.
4. Открыть меню **Конфигурация → Загрузить конфигурацию из файла**.
5. Выбрать файл:

   ```text
   configuration/УчебныйПрактикум.cf
   ```

6. Подтвердить загрузку.
7. Выполнить **Конфигурация → Обновить конфигурацию базы данных**.
8. Запустить базу в режиме **1С:Предприятие**.

## Вариант 2. Более правильный вариант для Git

Для дальнейшей разработки лучше выгрузить конфигурацию в файлы:

1. Открыть базу в Конфигураторе.
2. Перейти в **Конфигурация → Выгрузить конфигурацию в файлы**.
3. Выбрать папку `src/`.
4. Добавить папку `src/` в Git.

Так GitHub сможет показывать изменения в модулях, формах и метаданных. Файл `.cf` является бинарным, поэтому он удобен для загрузки проекта, но неудобен для анализа изменений.

## Почему `.dt` не добавлен

`.dt` — это выгрузка информационной базы. Она может содержать данные, пользователей, тестовые документы и другую информацию. Для публичного репозитория лучше не публиковать `.dt`, если нет уверенности, что внутри нет личных или лишних данных.
